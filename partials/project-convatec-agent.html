Key Components
  • RAG Lambda Orchestrator – Python 3.12 Lambda that receives patient queries, polls AppConfig for live parameters, invokes the Circuit Breaker, queries the Bedrock Knowledge Base, scores answer relevancy online, and publishes CloudWatch metrics per request.
  • Agentic System (FastAPI + LangGraph) – FastAPI server on AWS Lightsail hosting a LangGraph ReAct agent. Exposes a /chat endpoint; the agent reasons over tool results across multiple iterations (bounded by AppConfig agent_max_iterations).
  • RAG Tool (rag_search) – LangChain tool that the agent calls to retrieve and synthesise answers from the Bedrock Knowledge Base. Returns grounded answers with source attribution and relevancy scores.
  • MCP Server (FastMCP stdio) – FastMCP process launched as a stdio subprocess by the agent at startup. Exposes the assign_slot tool via the Model Context Protocol; bridges into DynamoDB with conditional writes to prevent double-booking.
  • AppConfig Client (token-based polling) – Polls AWS AppConfig on each Lambda cold start and every 30 seconds thereafter using a deployment token. Implements a three-tier fallback chain: new valid config → cached config → hardcoded defaults, so the system never fails from a missing config.
  • Circuit Breaker FSM – Tracks consecutive Bedrock KB failures; opens after 5 failures; enters HALF_OPEN after 30s to test recovery with 2 probe requests; closes on 2 consecutive successes. All thresholds tunable via AppConfig.
  • Hexagonal Ports / Adapters – RetrievalPort, EmbeddingPort, KnowledgeBasePort, and SlotRepositoryPort are Python Protocol interfaces. Concrete adapters (BedrockKBAdapter, TitanEmbeddingAdapter, DynamoDBAdapter) are injected at runtime; mock adapters are injected in tests.
  • Evaluation Pipeline – EventBridge weekly schedule invokes an evaluator Lambda that runs a RAGAS test suite (Faithfulness, Answer Relevancy, Context Precision, Context Recall) against a curated golden-set of Q&A pairs. Results are written to S3 and surfaced in a CloudWatch dashboard.
  • CloudWatch Metrics Layer – Custom namespace ConvatecRAG / AgenticSlotBooking; records query latency (p50, p90, p99), relevancy score, circuit breaker state, model ID used, and error rates. Billing alarm at $25/month.
Architecture Patterns

1) Adaptive Configuration via AWS AppConfig
All runtime decisions — model ID, temperature, max tokens, system prompt, retrieval top-k, search type, circuit breaker thresholds, and feature flags — are stored in AppConfig and polled with a token-based mechanism. A configuration change takes effect within seconds across all running Lambda instances with zero code deployment. This enables A/B testing of prompts, emergency model downgrade during traffic spikes, and maintenance-mode toggles via a single JSON update.

2) Circuit Breaker (Three-State FSM)
When the Bedrock Knowledge Base returns consecutive errors (default threshold: 5), the circuit breaker transitions from CLOSED to OPEN. In the OPEN state, all incoming requests are fast-failed with a friendly Spanish fallback message in < 500ms — no Bedrock calls, no patient-visible 60-second timeouts, no unnecessary billing. After a configurable recovery window (30s), the circuit enters HALF_OPEN and allows 2 probe requests. If both succeed, the circuit closes and normal operation resumes automatically.

3) Hexagonal Architecture / Ports & Adapters
The orchestrator depends only on abstract Python Protocol interfaces (ports), never on AWS SDK classes directly. Concrete adapters implement those ports and are injected at startup. This means the entire business logic layer can be unit-tested with moto mocks without hitting real AWS services, and the retrieval backend can be swapped from Bedrock KB to OpenSearch, Pinecone, or a local FAISS index without changing a single line of orchestration code.

4) Hierarchical PDF Chunking
Convatec InService documents are presentation-style PDFs with product diagrams, multi-column layouts, and mixed-language content. Standard PDF text extraction produces garbled output from diagrams. The ingestion pipeline uses Foundation Model parsing (Claude reads each page and converts diagrams to structured text descriptions), then applies parent chunks (1500 tokens) for full-section context and child chunks (500 tokens, 100-token overlap) for retrieval precision. This strategy produces dramatically better answers for image-heavy medical documentation.

5) Graceful Degradation Modes
The agentic system supports three degraded modes in addition to full operation: RAG-only (MCP server crashed — product Q&A still works), Slot-only (Bedrock KB unavailable — appointment booking still works), and Static fallback (both services down — circuit breaker OPEN serves a pre-written Spanish response instantly). Patients always receive a response, never an error page.

Agentic Flow (Request to Response)

1) Receive & validate: The patient sends a Spanish-language message to the /chat FastAPI endpoint. The request is validated (length, injection detection) and assigned a UUID for tracing. The LangGraph ReAct agent is invoked with the message and any prior conversation state.

2) ReAct reasoning & tool selection: The agent (Claude Haiku 4.5 via Bedrock) reasons about the user intent and selects one of two tools: rag_search for product questions ("¿Cómo se usa el catéter?") or assign_slot for appointment requests ("Quiero reservar una cita"). The agent may iterate across multiple reasoning steps (bounded by agent_max_iterations from AppConfig) to refine its answer.

3a) RAG path: rag_search calls the Lambda orchestrator, which checks the circuit breaker state. If CLOSED, it queries the Bedrock Knowledge Base (hybrid semantic + keyword search over S3 Vectors). Retrieved chunks are passed back to Claude for grounded answer generation. The answer is scored for relevancy (cosine similarity between query and answer embeddings via Amazon Titan Embeddings v2) and the score is published to CloudWatch.

3b) Booking path: assign_slot calls the FastMCP stdio subprocess, which writes a new slot to DynamoDB using a conditional expression (attribute_not_exists(slot_id)) to guarantee atomicity and prevent double-booking. The MCP server returns confirmation or a conflict message back to the agent.

4) Config & observability: Every parameter used in steps 2–3 — model ID, temperature, retrieval top-k, max iterations — was pulled from AppConfig at agent startup and can be changed between requests without redeployment. Every invocation publishes latency, circuit state, model used, and relevancy score to CloudWatch. Weekly, an EventBridge rule triggers a RAGAS evaluation against a golden Q&A set and stores results in S3.

Description

ConvaCare Agent is a proof-of-concept agentic assistant built for Convatec Colombia, a medical device company providing products for wound care, ostomy management, and continence care. Patients using these products often have specific, nuanced questions about usage, compatibility, and follow-up appointments — questions that require both accurate product knowledge and a frictionless scheduling experience.

The system combines two tools under a single LangGraph ReAct agent: a RAG tool that retrieves grounded answers from official InService product documentation, and an MCP-based slot-booking tool that reserves care appointments atomically in DynamoDB. The agent decides autonomously which tool to invoke (or both in sequence) based on the patient's message.

The architecture is explicitly aligned to the AWS Well-Architected Generative AI Lens across all six pillars. Reliability is addressed through a circuit breaker FSM and multi-mode graceful degradation. Operational excellence is achieved through AppConfig-driven runtime configuration and structured JSON logging. Performance is optimised by defaulting to Claude Haiku 4.5 (fast, cost-efficient) with on-demand escalation to Sonnet. Cost is minimised by using S3 Vectors (90% cheaper than OpenSearch), serverless Lambda for RAG, and Lightsail nano ($3.50/month) for the agentic layer. Sustainability and security round out the six pillars with 100% serverless compute and IAM least-privilege policies throughout.

The entire system is Spanish-first: system prompts, fallback messages, and the patient interface are all in Spanish. The RAG pipeline uses Foundation Model parsing to handle diagram-heavy PDF documents that defeat standard OCR-based extraction.

Tech stack: Amazon Bedrock (Claude Haiku 4.5, Claude 3.5 Sonnet), LangGraph, LangChain, FastAPI, FastMCP, langchain-mcp-adapters, Bedrock Knowledge Bases, Amazon Titan Embeddings v2, S3 Vectors, AWS AppConfig, DynamoDB, AWS Lambda, Amazon Lightsail, CloudWatch, EventBridge, AWS SAM, Python 3.12
