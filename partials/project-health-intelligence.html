Key Components
  • OCR Pipeline (LangGraph) – Invokes Amazon Textract, evaluates page-level confidence, stores output, and routes low-confidence documents (< 0.70) to the human review queue.
  • Extraction Pipeline (LangGraph) – Runs Comprehend Medical for English documents; invokes Claude Haiku 4.5 on Bedrock directly for Spanish / Portuguese documents. Links entities to RxNorm, ICD-10-CM, and SNOMED CT ontologies.
  • Reconciliation Engine (LangGraph) – Detects duplicate ClinicalFacts (same type + normalized value within a 7-day window), merges canonical records, flags conflicts for human review, and builds a longitudinal medication history.
  • Retrieval Pipeline (LangChain EnsembleRetriever) – Combines pgvector semantic search, PostgreSQL tsvector lexical search, and structured ClinicalFact lookup. Patient-scoped: patient_id filter is always applied first.
  • Safety Agent (LangGraph conditional node) – Classifies every query as allowed, allowed_with_caution, or not_allowed using a LangChain structured-output chain. Routes to the Answer Agent or a safe-refusal node.
  • Answer Agent (LangChain create_tool_calling_agent) – Generates grounded, cited answers using three tools: retrieve_evidence, lookup_clinical_facts, and cite_source. Conversation context persisted via LangGraph PostgreSQL checkpointer.
  • Bedrock Guardrails – Applied to both input and output to block PHI leakage, prompt injection, and prescriptive medical advice.
  • Human Review Queue – LangGraph interrupt mechanism pauses the pipeline for Trusted Reviewer corrections; resumes from checkpointed state after resolution.
  • Caregiver Consent – Patient-initiated invitation by email; delegated read-only or read-write access with optional expiration; revocation takes effect within 60 seconds.
  • FHIR R4 Export – Exports MedicationStatement, Condition, Procedure, Observation, AllergyIntolerance, and Encounter resources via Amazon HealthLake Patient/$everything.
  • Audit Log – Immutable, append-only log delivered to S3 with Object Lock; retained for six years per HIPAA requirements.

Q&A Flow (Safety → Retrieve → Answer → Cite)

1) Safety check: The Safety Agent classifies the query as allowed, allowed_with_caution, or not_allowed. Prescriptive advice (dosage changes, diagnoses, treatment selection) is refused; the agent returns what the documents say and suggests clinician questions instead.

2) Retrieve: The Retrieval Pipeline applies a patient_id filter first, then runs hybrid search combining pgvector semantic similarity and PostgreSQL tsvector lexical match. Structured ClinicalFact records are also included when the query matches a known entity type.

3) Answer: The Answer Agent (LangChain create_tool_calling_agent backed by Claude on Amazon Bedrock) generates a plain-language response using only the retrieved evidence. Every claim is cited using the cite_source tool (document, page, text span).

4) Guardrails: Bedrock Guardrails scan the output for PHI leakage, prompt injection, and unsafe medical content before the response is returned to the user.

5) Memory: Conversation state is persisted via a LangGraph PostgreSQL checkpointer, enabling follow-up questions and resumable sessions across devices without the patient repeating context.

Description

The Health Record Intelligence Platform is a HIPAA-aligned system on AWS designed for elderly patients and their family caregivers. It ingests unstructured medical documents — prescriptions, lab reports, discharge summaries, imaging results — and converts them into a structured, chronological health timeline with a grounded Q&A interface that cites every answer back to its source document.

The platform is built for a Spanish-speaking elderly audience from day one: Spanish and Portuguese documents are processed directly by Claude Haiku 4.5 on Amazon Bedrock without translation, and the entire UI and Q&A system is fully multilingual (English, Spanish, Portuguese).

The agentic layer is orchestrated with LangGraph: a Safety Agent classifies every query before the Answer Agent is invoked, ensuring the system never acts as a prescriber or diagnostician. Human-in-the-loop interruption points allow Trusted Reviewers to correct uncertain extractions before they reach the Timeline or retrieval index.

Infrastructure uses a two-tier CDK approach: dev/staging targets AWS Free Tier (Lambda + RDS PostgreSQL, simplified VPC), while production deploys ECS Fargate + Aurora Serverless v2 with full VPC isolation, KMS CMKs, AWS Config rules, and immutable CloudTrail logging.

Tech stack: Amazon Bedrock (Claude Haiku 4.5), Amazon Textract, Amazon Comprehend Medical, LangGraph, LangChain, Aurora PostgreSQL (pgvector + tsvector), Amazon HealthLake (FHIR R4), FastAPI, Amazon Cognito, Bedrock Guardrails, Bedrock Prompt Management, AWS CDK, Vite + TypeScript + MUI, ECS Fargate, Lambda, S3, KMS, CloudTrail, CloudWatch
